myFlix Application

myFlix Application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies

Allow users to remove a movie from their list of favorites
Request URL Method Request Body Response Body
Return a list of ALL movies to the user /movies GET None A JSON object holding data about all the movies
Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user /movies/:Title GET None A JSON object with movie title, cover art, description, genre, featured film, director
Return a list of ALL genres to the user /genres GET None A JSON object holding data about all the genres
Return data about a genre (description) by name/title /genres/:Name GET None A JSON object contain data about all the genres
Return a list of ALL directors to the user /directors GET None A JSON object holding data about all the directors
Return data about a director /directors/:Name GET None A JSON object contain data about director chose, bio, holding name, birthyear
Return a list of ALL users /users GET None A JSON object holding data about all the users
Return data about a single user by username /users/:Username GET None JSON object holding data about a single user by username
e.g.
{ "Username": "Username",
"Password": "Password",
"Email": "Email@Email.com",
"Birthday": "1990-01-01" }
Allow new users to register /users POST A JSON object holding data about the new user
{ Username: String,
Password: String,
Email: String,
Birthday: Date } 		A JSON object holding data about the user who registered
e.g.
{ "Username": "Username",
"Password": "Password",
"Email": "Email@Email.com",
"Birthday": "1990-01-01" }
Allow users to update their user info PUT /users/:Username A JSON object holding data about the users update
{ Username: String,
Password: String,
Email: String,
Birthday: Date } 		A response that contain the user´s updated information
e.g.
{ "Username": "Username",
"Password": "Password",
"Email": "Email@Email.com",
"Birthday": "1993-01-01" }
Allow existing users to deregister /users/:Username DELETE None A response that the account has been deleted
Allow users to add a movie to their list of favorites /users/:Username/movies/:_id POST A JSON object holding data about the movie added into favorite list A response containing that the user added a movie to the favorites list
e.g.
{ "FavoritMovies": [movieID],
"Username": "Username",
"Password": "Password",
"Email": "Email@Email.com",
"Birthday": "1990-01-01" }
/users/:Username/movies/:_id DELETE None A message indicating if the movie was successfully deleted
